# Objetivo

Registrar Architecture Decision Records e decisoes estruturais do MedSync.

# Escopo

Inclui templates e ADRs aprovados, propostos, rejeitados ou substituidos.

# Estrutura

- [ADR-0000.md](ADR-0000.md): template canonico.
- [ADR_TEMPLATE.md](ADR_TEMPLATE.md): template legado mantido.

# Fluxo

1. Copiar `ADR-0000.md`.
2. Nomear como `ADR-0001-titulo-curto.md`.
3. Preencher status, contexto, decisao, alternativas, consequencias e referencias.
4. Linkar documentos afetados.

# Boas práticas

- Criar ADR para decisoes com impacto duradouro.
- Nao registrar decisao sem contexto.
- Nao remover ADR historico; marcar como substituido.
