# Fluxo

1. Documentar apenas contratos existentes ou aprovados.
2. Linkar especificacoes e testes relacionados.
3. Atualizar autenticacao e autorizacao junto com seguranca.

# Boas práticas

- Nao criar endpoints ficticios.
- Nao publicar tokens, credenciais ou dados sensiveis.
- Marcar lacunas como `TODO`.
