# Objetivo

Definir o fluxo de revisao de codigo e documentacao no MedSync.

# Escopo

Bugs, regressao, seguranca, privacidade, testes, legibilidade e aderencia ao dominio.

# Responsabilidades

Revisores humanos e agentes devem priorizar riscos reais e evidencias verificaveis.

# Fluxo

1. Revisar diff e contexto.
2. Priorizar achados por severidade.
3. Conferir testes e documentacao.
4. Registrar perguntas abertas.
5. Aprovar somente quando riscos estiverem aceitaveis.

# Checklist

- Permissoes revisadas.
- Dados sensiveis protegidos.
- Testes coerentes.
- Links e docs atualizados.
- Sem claims indevidos de conformidade.
