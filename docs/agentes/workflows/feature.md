# Objetivo

Definir o fluxo para criar ou alterar funcionalidades no MedSync.

# Escopo

Discovery, especificacao, implementacao, testes, documentacao e entrega.

# Responsabilidades

Produto, engenharia, QA, seguranca e privacidade devem validar impacto antes da entrega.

# Fluxo

1. Confirmar objetivo e dominio.
2. Consultar documentos canonicos.
3. Mapear impacto em seguranca, LGPD, QA e B2B.
4. Implementar quando autorizado.
5. Atualizar documentacao e evidencias.

# Checklist

- Escopo claro.
- Riscos avaliados.
- Testes definidos.
- Documentos atualizados.
- Pendencias registradas.
