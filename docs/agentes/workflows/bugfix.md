# Objetivo

Definir o fluxo para correcao de bugs no MedSync.

# Escopo

Reproducao, causa raiz, correcao, regressao, evidencia e documentacao.

# Responsabilidades

Engenharia e QA devem corrigir sem ampliar escopo desnecessariamente.

# Fluxo

1. Reproduzir o problema.
2. Identificar causa raiz.
3. Corrigir com menor impacto seguro.
4. Adicionar ou atualizar teste.
5. Registrar evidencia.

# Checklist

- Bug reproduzido.
- Causa raiz descrita.
- Regressao coberta.
- Risco residual indicado.
- Documentacao afetada atualizada.
